Initial setting

const config = {
    baseURL: 'http://localhost:8080/',
    workspace: 'workspace-869f1a93'
} const api = new SemanticAPI(config)
First, to start working with the API, you must assign some configuration values that represent the base URL that is the address of the server where the data will be stored and the workspace where it will be stored. And then we can create a new SemanticAPI object that have functions to manipulate resources and datasets.

Mounting a resource

const resource = new Resource('contactData', 'contact', 'http://contactmail.com#Person')
resource.addVocabulary('schema', 'http://schema.org/')
resource.addVocabulary('vcard', 'http://www.w3.org/2006/vcard/ns#')
property = new Property('hasValue', 'email@email.com', false, 'email')
resource.addProperty('vcard', property)
property1 = new Property('client', 'YahooMail', false, 'email')
resource.addProperty('vcard', property1)
property2 = new Property('name', 'Lucimar da Silva Souza', false, '')
resource.addProperty('schema', property2)
property3 = new Property('about', 'Assunto do email', false, '')
resource.addProperty('schema', property3)
property4 = new Property ('telephone', '73991809696', false, '' )
resource.addProperty('schema', property4)
property5 = new Property('text', 'Mensagem do e-mail', false, '')
resource.addProperty('schema', property5)
resource.addCoordinatesDateTime()
resource.addLanguage('pt-br');
In this example is shown the assembly of a new resource.
First is used the function addVocabulary, from Resource, that have three the parameters: type, prefix and uri respectively
Then instances of Vocabulary object are created. To create one Vocabulary is necessary his prefix and namespace uri
Later are created new instances of Property. Each property has: name, value, a boolean that determines if the property will be stored as a Resouce (the uri of the object), and the last parameter determines the parent property. If this parameter is filled means that this property is actualy a property of a blank node belonging to the resource. Then is called the function addProperty of Resource class passing as parameters the vocab prefix and the property created early. The above example also show two functions addCoordinatesDateTime and addLanguage:
The first add the local, date and hour of where and when is being done the resource.
The second add a property that store the language of the resource added.

Saving a resource

api.saveResource(resource)
.then(json => alert(`Recurso salvo com sucesso\nSeu workspace é: ${json.workspace}
                    \nO ID do recurso é: ${json.resourceId}`);)
.catch(error => alert(`Problem trying save the resource: \n${error.message}`))
The saveResource function saves one resource later that it is mounted. This function belongs to the SemanticAPI class. Is used the object of this class created previously to call the function passing as parameter the resource created before. This function returns a promise to treat the cases of success or wrong in the operation. In the promise treatment events can be captured one object that have two variables: workspace and resourceId.

The getResource capture one resource given your uri. This function belongs to the SemanticAPI classe. Is used the object of this class created previously to call the function passing as parameter the resource created before. This function returns a promise to treat the cases of success or wrong in the operation. The resource searched can be captured in the promise treatment event.

})

Once captured one specific resource we can edit it adding vocabularies, properties, add a language or the coordinates and time. The resource searched can be captured in the promise treatment event. And then can be edited as in example above. But to make all changes permanent in dataset should be called the function commitChanges, belonging to the SemanticAPI class also. Should be given one special attention to the function addProperty wich in this case belong to SemanticAPI class but exits another way to add a new property calling the addProperty of Resource. In the first case the difference is that the first parameter should be the resource that will have the property added. Finnaly commitChanges is called to make the changes permanent in dataset.
